## 🎯 О проекте

VInput - это высокопроизводительный драйвер уровня ядра (Ring-0) для среды Windows, спроектированный для аппаратной эмуляции событий клавиатуры и мыши.

Проект разрабатывался как решение для автономных систем тестирования и автоматизации, где требуется обход программных ограничений пользовательского интерфейса (User-Mode) и независимость от физического оборудования ввода.

> Назначение проекта: автономные тестовые стенды, автоматизация без KVM-переключателей и физических USB-эмуляторов, интеграция с LLM-агентами, управляющими графическим интерфейсом через визуальную обратную связь.

Примечание: Исходный код данного проекта является коммерческой собственностью и не публикуется в открытом доступе. Данный репозиторий служит архитектурным портфолио, демонстрирующим навыки системной инженерии.

> 1 драйвер · Ring-0 уровень · Windows OS · 1 разработчик

## ⚡ Функциональность

### ⌨️ Эмуляция клавиатуры

Отправка кодов нажатия и отпускания клавиш. Обработка команд происходит на уровне системного драйвера обработки клавиатуры, что делает сигналы эквивалентными вводу с физической клавиатуры на уровне системных флагов.

### 🖱️ Управление мышью

Относительное и абсолютное позиционирование курсора с высокой точностью. Обработка состояний кнопок мыши и эмуляция прокрутки колеса через прямое взаимодействие с системным драйвером мыши.

Два профиля генерации ввода:

Adaptive / Human-like профиль - траектории курсора и тайминги нажатий генерируются с естественной вариативностью (микро-джиттер, нелинейные пути, неравномерные интервалы). Подходит для тестирования сценариев пользовательского интерфейса и интеграции с LLM-агентами.

Precision профиль - детерминированные, оптимальные команды (минимальные тайминги, кратчайшие траектории). Создан для высокоскоростных автоматизированных тестов и нагрузочных стендов.

## 🏗 Архитектура

Ключевые архитектурные решения:

Эмуляция на уровне ядра - драйвер взаимодействует напрямую со стеком обработки ввода, минуя ограничения User-Mode API. Генерирует события без программных меток источника ввода.

Динамическая адаптация - безопасный алгоритм динамического поиска и валидации точек входа в системные драйверы. Исключает использование жестко закодированных смещений памяти, обеспечивая совместимость от старых до современных сборок Windows.

Высокоскоростной обмен данными - взаимодействие между пользовательским приложением и драйвером организовано через буферизованный интерфейс.

## 🛠 Технологии

Язык: C / C++

Архитектура: Windows Driver Model (WDM)

Инструментарий: Windows Driver Kit (NTDDK), MSVC

Интерфейсы ОС: управление памятью на уровне ядра, взаимодействие с HID-стеком (драйверы мыши и клавиатуры)

## 🔐 Безопасность и стабильность

Управление уровнями прерываний - системные обработчики ввода требуют строгого контроля контекста выполнения. Драйвер безопасно управляет уровнями прерываний при передаче структур данных, исключая риск критических ошибок.

Изоляция данных - используемый метод обмена гарантирует, что система копирует данные во внутренний буфер ядра, защищая драйвер от изменения данных пользовательским процессом во время их обработки.

Валидация памяти - строгая проверка указателей при работе с памятью ядра.
